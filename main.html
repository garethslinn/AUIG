Introduction

Welcome to the Accessible User Interface Guidelines (AUIG). This guide is organised into three distinct categories: Global Settings, Elements, and Components. Each category addresses different aspects of accessibility to help you create inclusive and user-friendly interfaces. All categories are equally important and work together to ensure a comprehensive approach to accessibility.

1. Global Settings

The Global Settings section covers foundational configurations that affect the entire HTML document. This includes linking stylesheets, setting global ARIA attributes, and defining overall layout and theming options to ensure consistency and accessibility across all pages.

2. Elements

The Elements section focuses on making individual HTML elements accessible. This includes best practices for using semantic HTML, providing appropriate ARIA roles and labels, and ensuring that each element is usable by all users, including those relying on assistive technologies.

3. Components

The Components section addresses the accessibility of more complex interface elements such as modals, navigation menus, sliders, and other interactive components. Guidelines in this section ensure that these components are navigable, operable, and understandable for all users.
