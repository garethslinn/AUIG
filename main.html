1. Global Settings

Use this section for managing essential elements that apply across the entire web platform. It includes key aspects such as color, contrast, language, keyboard access, meta data, skip links, responsive design, and flashing images. When developing a new application or website or upgrading an existing system, prioritise global settings to ensure a consistent and effective user experience.

2. Elements

Use this section for managing specific UI components to ensure full accessibility. It covers best practices for elements such as multimedia (images, audio, video), forms, tables, and headings. Accessibility should be integrated into every phase of the project lifecycle, including design, user experience (UX), development, and quality assurance (QA). Prioritising these elements helps create an accessible and user-friendly application or website.

3. Components

Use this section for addressing the accessibility of complex UI elements. It covers components such as modals, tooltips, tabs, and accordions, providing essential guidelines and examples for effective implementation. While examples are included, users are encouraged to develop their own methods to achieve accessibility. The primary goal is to ensure each component is accessible and meets all relevant accessibility requirements.
