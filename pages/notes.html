1. Document Metadata

Elements that provide metadata about the HTML document, such as title, scripts, styles, and other information not displayed directly on the page.

  • <base>: Specifies the base URL for all relative URLs in the document.
  • <head>: Contains metadata and links to scripts, styles, etc.
  • <link>: Links external resources like stylesheets.
  • <meta>: Defines metadata like character set, viewport, and descriptions.
  • <style>: Embeds CSS styles within the document.
  • <title>: Sets the title of the document shown in the browser tab.

2. Content Sectioning

Elements that define the structure and layout of the document, helping to organise content into logical sections.

  • <address>: Provides contact information for the author/owner.
  • <article>: Represents a self-contained composition, such as a blog post.
  • <aside>: Contains content indirectly related to the main content, like sidebars.
  • <footer>: Defines the footer for a section or page.
  • <header>: Represents introductory content or navigation links.
  • <h1> to <h6>: Define headings, with <h1> being the highest level.
  • <main>: Specifies the main content of the document.
  • <nav>: Contains navigation links.
  • <section>: Groups related content into a thematic section.
  • <summary>: Provides a summary for <details>.
  • <details>: Creates a disclosure widget that can show/hide additional content.

3. Text Content

Elements that handle the main textual content of the document.

  • <blockquote>: Denotes a section quoted from another source.
  • <dd>: Description/details of a term in a definition list.
  • <div>: Generic container for flow content, used for styling and scripting.
  • <dl>: Definition list.
  • <dt>: Term/name in a definition list.
  • <figcaption>: Caption for a <figure>.
  • <figure>: Self-contained content, like images with captions.
  • <hr>: Represents a thematic break between paragraphs.
  • <li>: List item within <ul>, <ol>, or <menu>.
  • <ol>: Ordered list.
  • <p>: Paragraph.
  • <pre>: Preformatted text.
  • <ul>: Unordered list.

4. Inline Text Semantics

Elements that define the meaning and structure of text within a block of content.

  • <a>: Hyperlink to another page or location.
  • <abbr>: Abbreviation or acronym.
  • <b>: Bold text without extra importance.
  • <bdi>: Isolates a span of text to prevent it from being affected by surrounding text direction.
  • <bdo>: Overrides the current text direction.
  • <br>: Line break.
  • <cite>: Title of a work.
  • <code>: Inline code snippet.
  • <data>: Links content with machine-readable data.
  • <dfn>: Defines a term.
  • <em>: Emphasised text, typically rendered in italics.
  • <i>: Italic text without extra importance.
  • <kbd>: User input, typically rendered in a monospace font.
  • <mark>: Marked/highlighted text.
  • <meter>: Scalar measurement within a known range.
  • <noscript>: Content displayed when scripts are disabled.
  • <output>: Result of a calculation or user action.
  • <picture>: Container for multiple image sources.
  • <progress>: Displays progress of a task.
  • <q>: Short inline quotation.
  • <ruby>: Ruby annotations for East Asian typography.
  • <s>: Text that is no longer accurate or relevant.
  • <samp>: Sample output from a computer program.
  • <small>: Side comments or small print.
  • <span>: Generic inline container for text.
  • <strong>: Important text, typically rendered in bold.
  • <sub>: Subscript text.
  • <sup>: Superscript text.
  • <time>: Represents a specific time or date.
  • <u>: Underlined text without extra importance.
  • <var>: Variable in a mathematical expression or programming context.

5. Image and Multimedia

Elements that handle images, videos, audio, and other media types.

  • <area>: Defines an area within an image map.
  • <audio>: Embeds sound content.
  • <img>: Embeds an image.
  • <map>: Defines an image map with clickable areas.
  • <track>: Specifies text tracks for media elements like <audio> and <video>.
  • <video>: Embeds video content.

6. Embedded Content

Elements for embedding external resources like documents, iframes, and more.

  • <embed>: Embeds external content.
  • <iframe>: Embeds another HTML page within the current page.
  • <object>: Embeds external resources, such as images, videos, or Flash content.
  • <param>: Defines parameters for <object>.
  • <source>: Specifies multiple media resources for <audio> and <video>.

7. Scripting

Elements related to scripting and dynamic content.

  • <canvas>: Used to draw graphics on the fly via scripting.
  • <noscript>: Content to display if scripts are disabled.
  • <script>: Embeds or references executable scripts.
  • <template>: Holds client-side content that is not rendered immediately.
  • <slot>: Placeholder inside a web component for content.

8. Demarcating Edits

Elements that indicate changes to the document.

  • <del>: Text that has been deleted from the document.
  • <ins>: Text that has been inserted into the document.

9. Table Content

Elements for creating and managing tables.

  • <caption>: Provides a title for a table.
  • <col>: Specifies column properties for each column within a <colgroup>.
  • <colgroup>: Groups columns for styling.
  • <table>: Defines a table.
  • <tbody>: Groups the body content in a table.
  • <td>: Table cell containing data.
  • <tfoot>: Groups the footer content in a table.
  • <th>: Header cell in a table.
  • <thead>: Groups the header content in a table.
  • <tr>: Table row.

10. Forms

Elements that create interactive forms for user input.

  • <button>: Clickable button.
  • <datalist>: Provides a list of predefined options for an <input>.
  • <fieldset>: Groups related elements in a form.
  • <form>: Defines a form for user input.
  • <input>: Various types of input fields.
  • <label>: Defines a label for an <input>.
  • <legend>: Caption for a <fieldset>.
  • <meter>: Represents a scalar measurement.
  • <optgroup>: Groups related options within a <select>.
  • <option>: Defines an option in a <select> list.
  • <output>: Represents the result of a calculation.
  • <progress>: Displays progress of a task.
  • <select>: Drop-down list.
  • <textarea>: Multiline text input.

11. Interactive Elements

Elements that enhance user interaction beyond forms.

  • <details>: Creates a disclosure widget that users can open and close.
  • <dialog>: Represents a dialog box or window.
  • <menu>: Defines a list/menu of commands.
  • <menuitem>: Represents a command/menu item within a <menu>.
  • <summary>: Summary for a <details> element.

12. Web Components

Elements used to create custom, reusable components.

  • <slot>: Placeholder inside a web component for external content.
  • <template>: Holds client-side content that is not rendered immediately.

13. Obsolete Elements

Elements that are deprecated and should not be used in modern HTML. They may still work in browsers but lack support and accessibility features.

  • <acronym>: Use <abbr> instead.
  • <applet>: Use <object> instead.
  • <basefont>: Use CSS for font styling.
  • <big>: Use CSS for font size.
  • <blink>: Non-standard and unsupported.
  • <center>: Use CSS for alignment.
  • <font>: Use CSS for font styling.
  • <frame> and <frameset>: Use <iframe> instead.
  • <isindex>: Deprecated in favor of <input>.
  • <keygen>: Deprecated, use other authentication methods.
  • <listing>: Use <pre> instead.
  • <marquee>: Non-standard and unsupported.
  • <multicol>: Deprecated, use CSS columns.
  • <nextid>: Deprecated.
  • <nobr>: Use CSS white-space property.
  • <spacer>: Use CSS for spacing.
  • <strike>: Use <s> or <del> instead.
  • <tt>: Use CSS for monospace font.
  • <xmp>: Use <pre> instead.

Summary

Understanding and properly utilising HTML elements is crucial for creating well-structured, accessible, and semantically meaningful web pages. Here's a quick recap of the key categories:

  • Document Metadata: Information about the document, not displayed directly.
  • Content Sectioning: Organising content into logical sections.
  • Text Content: Managing the main textual content.
  • Inline Text Semantics: Enhancing the meaning of text within blocks.
  • Image and Multimedia: Handling visual and audio media.
  • Embedded Content: Incorporating external resources.
  • Scripting: Adding dynamic and interactive features.
  • Demarcating Edits: Indicating changes in content.
  • Table Content: Creating and managing tables.
  • Forms: Building interactive user input forms.
  • Interactive Elements: Enhancing user interaction beyond forms.
  • Web Components: Creating custom, reusable components.
  • Obsolete Elements: Deprecated tags to avoid in modern HTML.
